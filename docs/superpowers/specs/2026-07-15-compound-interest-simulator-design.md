# 복리 투자 시뮬레이터 — 설계 문서 (v1)

## 목적

개인 재무 계획용으로 사용할 복리 투자 시뮬레이션 사이트. 초기 투자금, 정기 납입, 수익률 등을 세세하게 입력받아 시간에 따른 자산 증가를 계산하고, 대시보드 형태로 결과를 보여준다.

## 범위

### v1 (이번 스펙)
- 세금 미반영 (이자소득세 등 계산하지 않음)
- 수익률: 고정 연 수익률 입력 + 연도별 개별 오버라이드
- 물가상승률 반영 (명목/실질 수익률 토글로 비교 가능)
- 구현: 단일 HTML 파일 (서버 없이 브라우저에서 바로 실행)
- 배포: 파일 형태로 공유 (호스팅 없음)

### v2 이후 (이번 스펙 범위 밖, 로드맵으로만 기록)
- 몬테카를로 방식의 랜덤 수익률 시뮬레이션 (평균/표준편차 입력 → 분포 결과)
- 낙관적/중립적/보수적 시나리오 비교
- 웹사이트로 정식 배포 (GitHub Pages, Vercel 등)

## 입력 변수

### 투자/납입 구조
| 변수 | 설명 |
|---|---|
| 초기 투자금 | 시작 시점의 일시 투자 금액 |
| 정기 납입액 | 매 주기마다 추가로 납입하는 금액 |
| 납입 주기 | 월 / 분기 / 연 중 선택 |
| 납입액 연 증가율 | 매년 납입액이 일정 비율로 증가 (예: 연봉 인상 반영) |
| 납입 중단/변경 시점 | 특정 연차부터 납입 금액을 중단하거나 변경 |

### 수익률/복리 구조
| 변수 | 설명 |
|---|---|
| 연 수익률 | 기본으로 적용되는 고정 연 수익률 |
| 복리 주기 | 월복리 / 연복리 등 이자가 원금에 합산되는 주기 |
| 연도별 수익률 오버라이드 | 결과 테이블에서 특정 연도의 수익률 셀을 직접 수정하면 해당 연도만 다른 수익률 적용, 이후 재계산 |
| 물가상승률 | 입력 시 명목 금액과 물가상승률을 반영한 실질 금액을 토글로 비교 |

### 표시 설정
- 화폐 단위: 만원 단위로 표시 (예: 1,000,000원 → 100만원)

## 계산 모델

- 기준 통화는 원화(KRW), 만원 단위로 반올림 없이 계산 후 표시만 만원 단위로 포맷
- 복리 계산은 설정된 복리 주기마다 수익을 원금에 합산하는 방식으로 진행
- 정기 납입은 설정된 납입 주기 시작 시점에 투입되는 것으로 가정 (기초 납입)
- 납입액 연 증가율은 매 1년 경과 시점에 적용
- 연도별 수익률 오버라이드가 있는 해는 해당 연도 전체 구간에 오버라이드 값을 적용하고, 오버라이드가 없는 해는 기본 연 수익률 적용
- 실질 수익률 토글 on 시: 각 연도 말 자산에서 물가상승률 누적치를 할인하여 실질 가치로 환산해 별도 표시

## 결과 화면 (레이아웃: 한 페이지 스크롤형)

입력 패널(좌측 데스크톱 / 하단 접이식 바텀시트 모바일) + 아래 내용이 순서대로 쌓이는 구조:

1. **요약 카드 + 자산 증가 그래프**: 최종 자산, 누적 납입 원금, 누적 수익금을 카드로 표시하고, 시간 경과에 따른 총자산 증가를 선 그래프로 표시. 명목/실질 토글 적용 시 그래프에 두 선(명목/실질) 또는 토글로 전환
2. **원금 vs 수익 비율**: 누적 납입 원금과 누적 수익금의 비율을 도넛 또는 막대 그래프로 표시 (복리 효과 체감용)
3. **연도별 상세 테이블**: 연도, 납입액, 해당 연도 수익금, 누적 원금, 누적 수익, 총자산 컬럼. 수익률 컬럼은 클릭하여 직접 값을 수정할 수 있으며, 수정 시 해당 연도부터 전체 재계산

## UI/UX

- **디자인 톤**: 데이터 중심 대시보드형
- **색상 테마**: 네이비 다크 기본, 라이트/다크 모드 토글 버튼 제공
- **입력 컨트롤**: 슬라이더 + 숫자 입력창 병행 (슬라이더로 대략 조정, 숫자창으로 정밀 입력)
- **모바일 대응**: `viewport` 메타 태그 필수 적용. 입력 패널은 데스크톱에서 사이드바, 모바일에서는 접히는 바텀시트/드로어로 전환. 반응형 CSS(미디어 쿼리)로 레이아웃 전환

## 기술 스택

- 단일 HTML 파일 (HTML+CSS+JS 인라인, 서버 불필요)
- 차트 라이브러리는 CDN으로 로드 (예: Chart.js) — 오프라인 완전 지원은 v1 범위 밖, 인터넷 연결 전제
- 계산 로직은 순수 JS로 구현 (프레임워크 불필요)

## 테스트 관점

- 복리 계산 정확성: 알려진 공식(예: 초기금만 있는 단순 복리)과 계산 결과 일치 여부 수동 검증
- 연도별 오버라이드 후 재계산이 이후 연도까지 올바르게 반영되는지 확인
- 명목/실질 토글 전환 시 물가상승률 계산이 올바른지 확인
- 모바일 화면(좁은 뷰포트)에서 바텀시트 개폐 및 레이아웃 붕괴 여부 확인
- 다크/라이트 모드 전환 시 대비(contrast)와 가독성 확인
